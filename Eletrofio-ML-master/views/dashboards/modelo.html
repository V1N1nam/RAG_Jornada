Feature Importance — Random Forest
Mostra o peso de cada variável de temperatura na decisão do modelo. Barras vermelhas (>25%) são dominantes — a variável mais influente para prever falhas. Um modelo bem calibrado distribui a importância entre 3 ou mais features: se uma única variável ultrapassar 50%, pode indicar overfitting nos dados de treino.

Distribuição de Scores
Proporção de compressores em cada faixa de risco. Ideal: maioria em Baixo (<40%). Se >30% estiver em Alto (>70%), o modelo pode estar descalibrado — ou existe um problema real sistémico na frota que merece inspecção física de uma amostra aleatória para validar.
Random Forest
Classifica o risco por aprendizagem supervisionada em dados históricos de temperatura. Mais estimadores = modelo mais robusto mas mais lento a inferir. O nº de features indica quantas variáveis de telemetria foram usadas no treino — mais features exigem mais dados para generalizar correctamente.

OneClass SVM
Aprende o padrão "normal" de operação e detecta desvios — útil para anomalias que nunca ocorreram no histórico de treino. O parâmetro nu controla a taxa esperada de falsos positivos: nu=0.1 significa que até 10% dos dados normais podem ser sinalizados como anomalia. Usado em conjunto com o RF reduz falsos negativos.
